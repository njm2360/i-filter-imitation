# i-filter-imitation

マルウェアスキャン・アクセスログ・ドメインブロックを行うMITMプロキシ
i-Filter風のコンテンツフィルタリングを自前で構築したい場合に使えます
MITMでSSLインスペクションを実装しているため、HTTPSでも検査が可能です(クライアントに証明書をインストール必須)

---

## 機能

| 機能                   | 説明                                                                                                           |
| ---------------------- | -------------------------------------------------------------------------------------------------------------- |
| **HTTPS 傍受**         | 動的に生成したリーフ証明書で TLS をターミネートし、上流への接続を再確立                                        |
| **マルウェアスキャン** | ClamAV (clamd) にファイルをストリーミングしてウイルス検査                                                      |
| **スマートデリバリ**   | ブラウザにはスキャン中 HTML を返しポーリングさせ、CLI ツールにはボディをトリクルしてスキャン完了まで待機させる |
| **ドメインブロック**   | テキストファイルによるブロックリスト（ワイルドカード `*.example.com` 対応）                                    |
| **アクセスログ**       | RFC 5424 構造化 syslog でリクエスト全情報を非同期送信                                                          |
| **証明書キャッシュ**   | Redis にリーフ証明書をキャッシュし再生成コストを削減                                                           |

## アーキテクチャ

```
Client
  │
  │  HTTP CONNECT / plain HTTP
  ▼
┌─────────────────────────────────┐
│                                 │
│  CONNECT ──► TLS 傍受          │
│                ├ ブロックリスト  │
│                ├ スキャン層     │──► clamd (ClamAV)
│                └ アクセスログ   │──► syslog サーバ
│                                 │
│  証明書キャッシュ ◄──► Redis    │
└─────────────────────────────────┘
  │
  ▼ 上流サーバへ転送
```

## 使用方法

### 1. CA証明書の生成

初回起動時に自動生成されます。生成された`proxy-ca.crt`をクライアント端末にインストールしてください。

```bash
docker compose up proxy   # 起動すると /data/ca/ に CA 証明書が生成される
```

### 2. サービス起動

```bash
docker compose up -d
```

### 3. プロキシ設定

端末やブラウザのプロキシを`http://<ホスト>:8080`に設定し、`proxy-ca.crt`を信頼済み CA としてインストールします。

## 環境変数

| 変数               | デフォルト              | 説明                          |
| ------------------ | ----------------------- | ----------------------------- |
| `LISTEN_ADDR`      | `:8080`                 | プロキシのリッスンアドレス    |
| `CA_CERT`          | `/data/ca/proxy-ca.crt` | CA証明書パス                  |
| `CA_KEY`           | `/data/ca/proxy-ca.key` | CA秘密鍵パス                  |
| `REDIS_URL`        | `redis://redis:6379/0`  | Redis接続 URL                 |
| `CLAMD_ADDR`       | `tcp://clamav:3310`     | clamdアドレス (`unix://`も可) |
| `SCAN_MAX_SIZE_MB` | `500`                   | スキャン上限サイズ (MB)       |
| `SCAN_TTL_MINUTES` | `60`                    | スキャンジョブの保持時間      |
| `SCAN_TEMP_DIR`    | `/tmp/scan`             | スキャン用一時ファイル置き場  |
| `BLOCK_LIST`       | (なし)                  | ブロックリストファイルパス    |
| `SYSLOG_ADDR`      | (なし)                  | syslog送信先 (`host:port`)    |
| `SYSLOG_PROTO`     | `udp`                   | `udp`または`tcp`              |
| `SYSLOG_QUEUE`     | `1000`                  | syslog送信バッファサイズ      |

## ブロックリスト

1行1ドメインのテキストファイル。`#` はコメント。ワイルドカード対応。

```
# 完全一致
example.com

# サブドメイン含む全てをブロック
*.example.com
```

## マルウェアスキャンの動作

### ブラウザ

1. ダウンロードを傍受し、バックグラウンドでスキャン開始
2. クライアントには即座にスキャン中 HTML を返す
3. ブラウザが `/scan/status?id=<jobID>` を 2 秒ごとにポーリング
4. **Clean** → ダウンロードボタンを表示 (`/scan/download?id=<jobID>`)
5. **Infected** → 脅威名とともに警告ページを表示

### CLIツール (`curl`, `wget` など)

1. ダウンロードを傍受し、同時にスキャン開始
2. スキャン完了まで約1B/sでボディをトリクル送信してコネクションを維持
3. **Clean** → 残りのボディを一気に送信
4. **Infected** → TCP コネクションを強制切断
